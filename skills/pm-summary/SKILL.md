# pm-summary — 产品决策汇总报告

## 执行流程

### Step 1：获取汇总模板
Read `C:\Users\Administrator\.claude\skills\pretend_as_PM\references\template.md` 第三节获取汇总报告模板。

### Step 2：解析时间范围

| 输入 | 含义 |
|------|------|
| 无参数 / "本月" | 当前自然月 |
| "上月" | 上一自然月 |
| Q1 / Q2 / Q3 / Q4 | 指定季度（当年） |
| YYYY-MM（如"2026-05"） | 指定月份 |
| "全部" / "所有" | 从文件创建至今全部 |

### Step 3：读取并筛选决策记录
1. 读取项目根目录 `pretend_as_PM.txt`
2. 按 `---` 分割解析各条决策记录
3. 根据 `【决策时间】` 字段筛选时间范围内的记录
4. 排除决策ID后缀为 `SUMM` 的汇总记录（避免汇总套汇总）
5. 提取每条记录关键字段：ID、时间、模块、重要级、选型结论、收益、未解决问题

### Step 4：生成汇总报告
按模板生成结构化报告，包含：
- **决策概览**：总数、重要级分布、模块数
- **按模块分布**：表格列出各模块决策数和关键摘要
- **核心决策回顾**：最重要的3-5条，2-3句话简述
- **落地收益汇总**：量化和定性收益按模块归类
- **PM思考与反思**：决策模式分析、反复决策检查、验证结果回顾、后续方向建议
- **未解决问题跟踪**：汇总所有未解决问题及当前状态

### Step 5：写入并展示
1. 决策ID：`PM-YYYYMMDD-HHMMSS-SUMM`
2. 追加写入 `pretend_as_PM.txt`
3. 对话中展示完整报告
4. 确认写入成功

### 注意事项
- `pretend_as_PM.txt` 不存在或时间范围内无记录 → 如实告知，不生成空报告
- PM思考与反思应诚实、有洞察力，不写正确的废话
- 汇总报告本身是特殊决策记录，ID后缀 `SUMM` 区别于普通决策
