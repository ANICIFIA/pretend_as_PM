# pretend_as_PM — Claude Code 产品经理视角决策记录

让 Claude Code 在开发过程中以产品经理视角自动记录每一个方案抉择，生成可追溯、可用于项目复盘的决策日志。

## 核心功能

- **自动触发**：检测到方案抉择时自动记录决策到 `pretend_as_PM.txt`
- **手动命令**：`/pm-decision` 手动记录决策，`/pm-summary` 生成月度/季度汇总报告
- **决策搜索**：新决策前自动搜索历史记录，避免重复讨论
- **决策撤回**：方案被推翻时追加撤回记录而非删除历史
- **声明机制**：每次决策后输出 `📋 决策已记录：PM-xxx`，漏记一目了然
- **三级重要性**：🔴重要 / 🟡一般 / 🟢轻量，不同级别不同填写要求

## 文件结构

## 安装

### 自动安装

**macOS / Linux：**
```bash
chmod +x install.sh && ./install.sh
```

**Windows：**
```cmd
install.bat
```

### 手动安装

将 `skills/` 目录下的三个文件夹复制到 Claude Code 的全局 skills 目录：

- **macOS / Linux**：`~/.claude/skills/`
- **Windows**：`C:\Users\<用户名>\.claude\skills\`

### 强化触发（推荐）

在全局 CLAUDE.md 中添加以下行，确保每轮对话都能看到提醒：

```markdown
- 涉及方案抉择时，触发 pretend_as_PM Skill 记录决策后再修改代码。
  修改完成后输出 `📋 决策已记录：PM-xxx`（无方案抉择则无需输出）。
  若用户发现应记录但未记录，可手动 `/pm-decision`。
```

**macOS / Linux**：`~/.claude/CLAUDE.md`
**Windows**：`C:\Users\<用户名>\.claude\CLAUDE.md`
**项目级**：`<项目根目录>/.claude/CLAUDE.md`

## 使用方式

| 场景 | 方式 |
|------|------|
| 开发中遇到方案抉择 | Claude 自动触发，记录后输出 `📋` |
| 手动补记一个决策 | 输入 `/pm-decision` 或说"记录决策" |
| 查看月度决策汇总 | 输入 `/pm-summary` 或"本月决策汇总" |
| 查看上季度汇总 | `/pm-summary Q1` |
| 决策被推翻 | Claude 自动追加撤回记录，引用原决策 ID |

## 决策记录示例

```
【决策ID】：PM-20260531-143052-0001
【决策时间】：2026-05-31 14:30:52 (UTC+8)
【决策重要级】：🔴重要
【决策模块】：Agent用户意图识别
【决策背景】：原有Prompt内配置大量意图识别约束规则（约800字）...
【备选方案罗列】
方案A（保守精简）：保留规则框架，删减冗余 → 风险低但根本问题未解决
方案B（激进放手）：删除全部限制，仅保留输出格式 → Prompt缩减85%但需验证准确率
方案C（混合过渡）：保留核心3条 → 折中但判断主观
【产品选型结论】：选用方案B
【选型核心依据（PM视角）】：...
【落地收益】：token降低约85%，后续零维护成本
【未解决问题】：极端case下意图识别漂移风险
【后续验证计划】：A/B测试 + 灰度发布
```

## 设计哲学

- **技能服务于项目，而非反过来**：简单方案胜出就是好决策，不为记录而复杂化
- **诚实 > 好看**：不确定就说需上线观测，不适用就标不适用
- **因地制宜**：最适合当前项目、阶段、场景的方案就是最好的方案
